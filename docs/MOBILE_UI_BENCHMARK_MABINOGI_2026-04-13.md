# MOBILE UI/UX Benchmark — Mabinogi Style (2026-04-13)

## 방향 결정
SOOM 모바일 UI/UX는 당분간 `마비노기 스타일`을 벤치마크 기준으로 삼는다.

## 핵심 원칙
1. 따뜻한 판타지 톤 (차가운 SaaS 톤 금지)
2. 기능보다 분위기 먼저 인지되게 구성
3. 카드/버튼은 딱딱한 관리도구 느낌보다 손맛 있는 오브젝트 느낌
4. 텍스트는 짧고 행동 중심
5. 복잡한 HUD 금지, 필요한 정보만 레이어로 노출

## 비주얼 가이드
- 배경: 딥 네이비 + 안개톤 오버레이
- 포인트 색: 골드/아이보리/민트(강한 원색 최소화)
- 카드: 양피지 느낌(밝은 베이지 계열), 얇은 테두리
- 버튼: 둥근 캡슐 + 은은한 그림자
- 타이포: 제목은 감성, 본문은 명확

## UX 가이드
- 월드 탭: 지도/오브젝트 중심 탐색
- 채팅 탭: 명령 입력 + 즉시 실행 항목 생성
- 할 일 탭: 실행 대기/완료 분리, 터치로 빠른 상태 전환
- 사람 탭: 상태와 다음 행동만 우선 노출

## 적용 순서
1) 월드 탭 리스킨
2) 채팅/할 일 컴포넌트 톤 통일
3) 공통 스타일 토큰 분리
4) 이미지/에셋 본제작
