# SOOM WORLD GAME LAYER - 2026-04-10

## 한 줄 정의
숨은 `채팅으로 운영하고, 월드로 목장을 보는 게임형 운영 OS`다.

---

## 핵심 원칙
- 채팅은 계속 가장 쉬운 입력 인터페이스다.
- 월드는 상태를 보여주는 메인 시각화 레이어다.
- People / Households / Timeline은 상세 데이터 패널이다.
- 목표는 게임화가 아니라 `공간형 운영 시각화`다.
- 점수, 경쟁, 레벨업 같은 장난감 요소는 넣지 않는다.

---

## 제품 구조

### 1. Chat = 조작창
- 목자는 채팅으로 새가족 등록, 출석, 심방, 기도제목, 후속조치를 입력한다.
- 채팅은 명령창처럼 작동한다.

### 2. World = 메인 화면
- 목장 전체를 하나의 작은 마을처럼 본다.
- 목자 = 플레이어 캐릭터
- 목원 = NPC/미니 캐릭터
- 가정 = 집
- 교회 이벤트 = 광장/예배당/공용 공간 이벤트

### 3. Detail Panels = 데이터 확인창
- People: 사람 상세 기록
- Households: 가정 구조와 관계
- Timeline: 최근 반영 흐름
- Search: 빠른 복구/검색

---

## 월드 오브젝트

### 목자
- 대표 캐릭터 1명
- 현재 관리 중인 사람/가정을 향하는 포커스가 보이면 좋다.

### 목원
- 각 사람은 미니 캐릭터로 보인다.
- 리스트의 이름 텍스트보다 먼저 존재감이 보여야 한다.

### 가정
- 같은 가정은 같은 집/구역에 배치한다.
- 집을 클릭하면 가정 구조와 최근 기록을 본다.

### 목장 전체
- 하나의 작은 마을 구조
- 집, 길, 광장, 교회/모임 공간이 있는 형태를 기본으로 삼는다.

---

## 상태 이펙트

### 사람 단위
- 기도제목 있음 → 보라빛 반짝임
- 심방 필요 → 노란 느낌표
- 후속 연락 필요 → 편지/벨 아이콘
- 건강 이슈 → 파란 회복 이펙트
- 최근 좋은 변화 → 따뜻한 빛
- 새가족 후보 → 반투명 캐릭터 또는 `새 방문` 표식

### 가정 단위
- 가정 전체 기도 필요 → 집 위 빛/별 이펙트
- 관계 연결 미완성 → 점선/미완성 연결선
- 최근 활동 많음 → 창문 불빛, 메모 흔적

## 채팅과 월드의 연결

### 예시 1
입력:
`조성진 형제 오늘 처음 오셨습니다.`

결과:
- Intake candidate 생성
- 월드에 새 방문자 캐릭터 등장
- People 상단에 등록 후보 표시

### 예시 2
입력:
`다음 주에 다시 연락드리면 좋겠습니다.`

결과:
- 해당 사람 위에 후속 아이콘 생성
- Follow-up 성격 상태 부여

### 예시 3
입력:
`허리 아프셔서 병원 가셨어요.`

결과:
- 건강/돌봄 기록 생성
- 캐릭터 또는 집에 건강 이펙트 부여

즉 구조는 항상 이렇다.

`채팅 입력 -> 구조화 -> 운영 데이터 반영 -> 월드 상태 변화`

## 1차 화면 구조

### 기본 레이아웃
- 중앙: 월드 맵
- 하단 또는 좌하단: 채팅 입력창
- 우측: 선택한 사람/가정 상세 패널

### 핵심 UX
- 채팅은 최소형으로 유지
- 월드는 크게 보여야 함
- 상세는 클릭했을 때만 깊게 열림

---

## 구현 단계

### Step 1
- `/app/gido/world` 신규 페이지
- 픽셀풍 또는 2D 미니 월드 배경
- 실제 member / household 데이터를 읽어 기본 배치
- 상태를 배지보다 월드 이펙트로 표현

### Step 2
- 사람 클릭 → 우측 상세 패널
- 집 클릭 → 가정 상세 패널
- 등록 후보/후속/기도/건강 상태를 월드에서 직접 확인 가능하게 함

### Step 3
- 채팅 입력 직후 월드가 즉시 바뀌는 연결 강화
- 방문 뷰, 집 내부 뷰, 관계선 보강

---

## 하지 말 것
- 단순 오른쪽 요약 카드 수준에서 월드를 끝내지 말 것
- 이름만 반복 나열하는 UI로 회귀하지 말 것
- 과한 게임 요소(점수, 보상, 경쟁)를 넣지 말 것
- 운영 데이터보다 장식이 앞서지 말 것

---

## 현재 판단
선용이 원하는 월드는 `상태 카드`가 아니라 `목장 운영을 하나의 게임처럼 보는 공간형 화면`이다.

따라서 다음 구현은
- 채팅 UI 미세수정보다
- `실데이터가 붙는 world 화면 1차`를 먼저 만드는 게 맞다.
