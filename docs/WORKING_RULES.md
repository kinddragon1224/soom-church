# SOOM WORKING RULES

이 문서는 soom-church 작업 시 중간 승인 없이 연속 진행하기 위한 기준이다.

## 기본 원칙
- gloo workspace를 벤치마킹하되, SOOM 브랜드에 맞게 번역한다.
- workspace 개발은 PC 웹 기준으로 먼저 진행한다.
- 홈은 랜딩, workspace는 앱/대시보드처럼 유지한다.
- 사용자 노출 문구는 존댓말, 대화는 반말로 유지한다.
- 브랜드 컬러는 네이비 + 골드 기준으로 유지한다.

## 자율 진행 범위
- workspace UI/UX 개선
- 정보구조 정리
- 페이지 제목/카피 조정
- 컴포넌트/레이아웃/상태 카드/탭/보드/리스트 개선
- 빌드 통과 시 커밋/푸시까지 연속 진행

## 반드시 확인이 필요한 경우
- 브랜드 방향 변경
- 컬러 시스템 변경
- 요금제/결제 정책 변경
- 홈/워크스페이스의 큰 IA 전환
- 실제 개인정보/외부 서비스 연동/권한/보안 관련 변경

## 응답 규칙
- 불필요한 중간 보고를 줄인다.
- 사용자 보고는 짧고 실행 중심으로 한다.
- 가능하면 10줄 미만으로 요약 보고한다.
- 괜한 장문 설명보다 실제 개발 진행을 우선한다.
