## LATEST SESSION RECOVERY - 2026-04-11
### 현재 제품 정의
- 숨은 `목장 운영 개발 OS`다
- 기존 `gido`/기도 목장 베타는 폐기했고, 새 기본 워크스페이스는 `/app/beta`다
- 핵심 구조는 `운영 입력 -> 구조화 -> 운영 상태판`이고, 월드는 왼쪽 패널을 제외한 거의 전체 화면을 먹는 메인 장면이어야 한다
- 사용자가 원하는 감성 기준은 단순 픽셀 맵이나 카드형 요약이 아니라, 더 높은 퀄리티의 감성 장면/월드다

### 이번 세션 결정
1. `gido`는 사실상 삭제 상태로 보고 더 이상 메인 트랙으로 다루지 않는다
2. 새 beta는 `아이디 + 이름 + 비밀번호` 기반 간단 로그인으로 바로 진입한다
3. 월드는 `장면 우선`, `풀캔버스`, `에셋 기반`, `UI 오버레이 최소화` 원칙으로 간다
4. 리빌드는 한 번에 많이 얹지 않고 단계별로 검증한다

### 이번 세션 반영
- 운영 DB에서 `gido` 교회/데이터/멤버십/테스트 계정을 삭제함
- `gido` 전용 간편 로그인과 특수 진입 제거
- 새 beta 회원가입/로그인 흐름 추가
- 로그인/가입 후 기본 진입을 `/app/beta`로 연결
- `/app/beta`, `/app/beta/chat`, `/app/beta/world`, `/app/beta/records`, `/app/beta/settings` 뼈대 생성
- beta 홈을 운영 개발 OS 런처형 화면으로 정리
- world는 카드형/박스형 시도를 거쳐 풀캔버스 + 키비주얼 배경 + 최소 오버레이 구조까지 진행함
- 집/교회/캐릭터 에셋 생성과 적용까지 시도했지만, 그대로 올리면 스티커처럼 떠 보여서 배경 중심 장면 우선 원칙을 재확인함
- 현재 world는 장면 우선 구조로 다시 정리된 상태이며, 감성 톤 진입까지는 성공했지만 제품 퀄리티는 아직 미완성임

### 바로 다음에 할 것
1. 월드 배경 톤을 2~3장 더 생성해 메인 기준 장면을 더 고정하기
2. 상태 이펙트 전용 에셋(기도/후속/돌봄/새 방문) 고급화하기
3. 캐릭터/NPC를 진짜 스프라이트 느낌으로 다시 만들기
4. 그 다음에야 실제 데이터 연결을 붙이기

### 멈출 것
- div 박스/카드 조립만으로 월드 퀄리티를 올리려는 시도
- 집/교회/캐릭터를 배경 위에 그대로 스티커처럼 얹는 방식
- 설명 패널이 장면보다 먼저 읽히는 구조

### 운영 메모
- OpenClaw cron은 전부 제거된 상태 유지
- 텔레그램 발신은 정상
- `channels.telegram.apiRoot` 미지원 경고 로그는 나중에 설정 정리 필요
- 커밋 시 `app/workspace/page.tsx` 같은 무관한 로컬 변경은 계속 제외할 것
