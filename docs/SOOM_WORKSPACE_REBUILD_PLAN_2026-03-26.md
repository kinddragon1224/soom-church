# SOOM Workspace Rebuild Plan — 2026-03-26

## 1. 목적

이 문서는 `SOOM_VISUAL_MVP_STRUCTURE_2026-03-26.md`를 실제 제품 리빌드 작업으로 연결하기 위한 실행 계획이다.

핵심 목표:
- 현재 워크스페이스를 시각화 중심 구조로 재정렬한다.
- 대시보드형 홈을 구조 홈으로 전환한다.
- 사람 / 기록 / 공지 흐름을 시각화 홈 기준으로 다시 연결한다.

---

## 2. 리빌드 원칙

1. 기존 화면을 조금씩 다듬는 것이 아니라 정보 구조를 재배치한다.
2. 첫 화면은 숫자 카드보다 구조 시각화가 먼저다.
3. 부서/사역은 메인 트리가 아니라 오버레이 정보로 둔다.
4. 기록과 후속관리는 사람 구조와 직접 연결한다.
5. MVP는 100명 전후 소형교회에 최적화한다.

---

## 3. 작업 우선순위

## Phase 1 — 정보 구조 재배치
- [ ] 메인 메뉴를 구조 홈 중심으로 재정렬
- [ ] `홈`을 `구조 홈` 개념으로 재정의
- [ ] 사람 / 기록 / 공지 / 설정 우선순위 재조정

## Phase 2 — 구조 홈 v1
- [ ] 교구/목장 시각화 영역 배치
- [ ] 목장별 인원/후속/심방 상태 카드 연결
- [ ] 선택한 목장 상세 패널 배치
- [ ] 최근 기록/후속 대상 연결

## Phase 3 — 사람 상세 연결 강화
- [ ] 개인 상세에서 소속 구조를 더 선명하게 표현
- [ ] 가족 / 목장 / 교구 관계를 상단에 노출
- [ ] 기록과 다음 액션을 상세에 직접 연결

## Phase 4 — 기록 모듈 정리
- [ ] 기록 유형을 심방 / 상담 / 사역 / 운영으로 재정리
- [ ] 사람 상세에서 기록 진입 강화
- [ ] 기록 허브를 구조 기반 진입으로 개선

## Phase 5 — 공지 연결
- [ ] 공지 대상에 교구/목장 구조 연결
- [ ] 시각화 홈에서 공지 상태가 보이게 연결

## 5. 현재 결정 사항

- MVP의 메인 구조는 `교구 → 목장 → 가족 → 개인`
- 부서/사역은 메인 트리가 아니라 오버레이
- 홈은 기존 대시보드가 아니라 구조 시각화 중심
- 초기 타깃은 100명 전후 소형교회
- 차별점은 리스트 관리가 아니라 교적 구조 시각화
