# SOOM Mobile App Split Plan — 2026-04-13

## 결정
- 제품 본체는 웹 워크스페이스가 아니라 세로형 모바일 앱으로 간다.
- 웹은 `공개 홈페이지 + 로그인 + 관리자 콘솔`만 남긴다.
- 구형 workspace/dashboard 흐름은 더 이상 확장하지 않는다.

## 1. 영역 재분류

### 웹 홈페이지
남김
- `/`
- `/about`
- `/pricing`
- `/contact`
- `/login`
- `/signup`
- `/app/mobile` (앱 진입/설명 브리지)

역할
- 제품 소개
- 도입 문의
- 로그인 진입
- 앱 다운로드/앱 진입 연결

### 모바일 앱 본체
새 기준
- 월드
- 채팅
- 사람
- 일정/할 일
- 온보딩

역할
- 실제 목장 운영
- 시각화된 월드
- 채팅형 명령/기록
- 사람/가정/기도/돌봄 흐름

### 관리자 콘솔
남김
- `/platform-admin/*`

역할
- 내부 운영
- 고객/교회/계정/구독 관리
- 사용자 제품 경험과 분리된 백오피스

## 2. 모바일 앱 IA 초안

### 탭 구조
1. 월드
2. 채팅
3. 사람
4. 할 일

### 핵심 플로우
1. 로그인
2. 내 공동체 확인 또는 생성
3. 월드 진입
4. 채팅으로 기록/명령
5. 사람/가정/할 일 확인

### 세로모드 원칙
- 세로 전용 설계
- 한 손 사용
- 월드는 상태를 보여주는 화면
- 실행은 채팅/탭 중심
- 깊은 메뉴 구조 금지

## 3. 기술 분리

### 웹
- Next.js 유지
- 역할: 홈페이지 + 로그인 + 관리자 콘솔

### 모바일 앱
- React Native + Expo
- iOS / Android 동시 대응
- 기존 인증/DB/API 최대 재사용

### 공통
- Prisma/Postgres 유지
- Next API는 우선 공용 백엔드로 사용
- 필요 시 추후 API 서버 분리

## 4. 바로 진행할 작업
1. 앱 폴더 `apps/mobile-app` 생성
2. Expo 초기 뼈대 추가
3. 모바일 앱 IA 문서 기준으로 첫 화면 설계
4. 웹의 구형 workspace 흐름은 `/app/mobile` 또는 로그인으로 리다이렉트

## 5. 보류 원칙
- 구형 dashboard/graph workspace 고급화 보류
- platform-admin 디자인 개선 보류
- 웹에서 제품 본체처럼 보이는 새 화면 추가 보류

## 6. 다음 구현 우선순위
1. Expo 앱 부트스트랩
2. 로그인 화면
3. 모바일 홈/월드 탭 셸
4. 채팅 탭 1차
5. 사람 탭 1차
6. 온보딩 연결
