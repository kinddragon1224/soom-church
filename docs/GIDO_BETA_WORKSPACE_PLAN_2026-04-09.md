# G.I.D.O BETA WORKSPACE PLAN - 2026-04-09

## 제품 한 줄
숨은 목자가 목원을 더 정확하고 덜 피곤하게 관리하게 만드는 워크스페이스다.

## 지금 기준
- 첫 베타는 `우리 목장`, `G.I.D.O 목장`
- 홈은 launcher / overview
- 실제 운영은 `목원 관리`, `관리`, `중보`, `근황` 각 화면에서 처리
- AI는 장식이 아니라 정리, 복구, 우선순위 제안, 다음 행동 연결 역할

## 판단 기준
1. 목자가 덜 헤매는가
2. 객체가 분명한가
   - 목원
   - 가정
   - 기록
   - 출석
   - 교회 이벤트
   - 소속 / 부서
3. 화면 책임이 섞이지 않는가
4. 기록이 쌓였을 때 다음 행동으로 이어지는가
5. 예외를 덧대는가, 아니면 본질에 가까워지는가

## Anna Karenina 원칙을 숨에 적용하면
- 잘 작동하는 워크스페이스들은 핵심 구조가 서로 닮는다.
- 망가지는 제품은 각기 다른 예외와 누락 때문에 제각각 무너진다.
- 따라서 숨은 특이한 기능보다 잘 작동하는 제품들의 공통 구조를 먼저 닮아야 한다.

## 화면 책임
### 1. 홈
역할: 시작점, 브리프, 빠른 진입

포함
- 오늘 시작하기
- 업데이트 / 알림
- 주요 워크스페이스 바로가기
- 고정 목원
- 오늘 상황

제외
- 상세 편집 폼 다수
- 가정 운영 상세
- 기록 입력 중심 UI

### 2. 목원 관리
역할: 사람 운영의 메인 화면

포함
- 목원 추가
- 가정 연결 보완
- 관리 필요 대상 확인
- 상세 기록 바로가기
- 출석, 이벤트, 소속 관리로 진입

제외
- 홈 성격의 카드 과다 배치
- 가정 전체 운영 보드
- 중보 순서 자체 편집 중심 UI

### 3. 가정
역할: household context 관리

포함
- 가정 구성
- 배우자 / 자녀 / 가족 구분
- 중보 기도
- 연락 메모
- 가정 단위 흐름

### 4. 관리
역할: 오늘 처리할 대상과 운영 큐

포함
- 관리 필요 대상
- 리더 / 순환 진행 확인
- 기록 카드
- 오늘 처리 우선순위

## 목원 관리 상세의 기준 구조
- 기본 정보
- 가정 / 관계
- 상세 기록 / 메모
- 출석 관리
- 교회 이벤트
- 교구 / 목장 / 부서
- 사역 기록

## 현재 완료
- 홈 launcher화
- `목원` -> `목원 관리` 라벨 정리
- 간단 목원 추가
- 가정 안에서 아이 추가
- 목원 상세에 교회 이벤트 / 부서 / 사역 기록 추가

## 다음 우선순위
1. `목원 관리` 목록 화면에서 홈 성격 제거
2. `목원 관리`를 등록 / 보완 / 관리 / 기록 진입 중심으로 재구성
3. 가정 화면에서 배우자 / 자녀 / 기타 가족 구조를 더 선명하게 정리
4. 교회 이벤트를 현재 faith milestone 범위를 넘어서 일반 이벤트까지 확장할지 모델 검토
5. 다른 목장 목자들이 구글 로그인으로 자기 워크스페이스를 갖는 멀티 워크스페이스 구조 설계

## 구현 원칙
- 한 번에 한 화면씩 고친다
- 메모를 먼저 고정하고 구현한다
- 빌드 확인 후 커밋 / 푸시한다
- 목자가 실제로 어떻게 클릭하고 기록할지 기준으로 본다
