# SOOM V2 IA - 2026-04-09

## 최상위 구조
1. Today
2. Review
3. People
4. Households
5. Timeline
6. Search

핵심 원칙은 단순하다.
- Today는 오늘 처리
- Review는 AI 검토
- People은 사람 레코드
- Households는 가정 관계
- Timeline은 기록 흐름
- Search는 복구

## 1. Today
### 역할
오늘 목자가 바로 처리할 것만 보여준다.

### 포함
- 오늘 새로 들어온 변화
- 검토 대기 개수
- 바로 연락할 사람
- 최근 가정 변화
- 오늘 후속 3~5개

### 제외
- 대규모 입력 폼
- 전체 사람 목록
- 가정 구조 편집 상세

## 2. Review
### 역할
채팅에서 추출된 운영 데이터 후보를 확인하고 승인한다.

### 카드 종류
- 사람 매칭 불확실
- 가정 연결 불확실
- 관계 추론 불확실
- 기도제목 추출
- 심방/상담 기록 추출
- 출석 변화 감지
- 이벤트 기록 후보

### 카드 액션
- 승인
- 수정 후 승인
- 보류
- 무시
- 기존 기록과 병합

## 3. People
### 역할
사람 한 명의 운영 상태를 가장 빨리 파악하는 레코드 화면

### 목록 화면 포함
- 이름
- 가정
- 최근 변화 한 줄
- 상태
- 다음 액션

### 상세 화면 포함
- 기본 정보
- 최근 변화
- 관계
- 가정
- 기록 타임라인
- 다음 액션

### 제외
- 긴 입력폼 우선 구조
- 같은 이름을 여러 섹션에서 반복 노출하는 방식

## 4. Households
### 역할
가정 구조와 관계를 정리하는 화면

### 포함
- 가정 대표
- 배우자
- 자녀
- 기타 가족
- household 단위 기도/메모
- household 관계 보기

### 핵심 표현
- 이름 나열보다 관계선, 역할, 최근 변화 중심

## 5. Timeline
### 역할
목장에서 일어난 일을 시간 흐름으로 본다.

### 포함
- 심방
- 상담
- 출석
- 교회 이벤트
- 기도제목 변화
- 후속조치

### 뷰 기준
- 날짜
- 사람
- 가정
- 유형
- 처리 상태

## 6. Search
### 역할
빠르게 복구한다.

### 질의 예시
- 지난주 결석한 사람
- 최근 아픈 사람
- 새가정 A와 연결된 사람
- 최근 심방 기록
- 배우자 연결 안 된 가정

## 첫 화면 와이어프레임 텍스트
### Today
- 상단: 오늘 새 변화 수 / 검토 대기 수 / 바로 연락할 사람 수
- 본문 좌측: 오늘 들어온 변화 피드
- 본문 우측: 검토 대기 카드 5개
- 하단: 바로 연락할 사람, 최근 가정 변화

### Review
- 상단 필터: 전체 / 사람 / 가정 / 관계 / 기록 / 출석 / 이벤트
- 본문: review card list
- 우측 패널: 선택 카드의 추출 원문, 제안 구조, 승인 액션

### People
- 좌측: 사람 리스트
- 우측: 선택한 사람 요약 레코드
- 중요한 것만 보이고, 수정은 필요한 곳만 펼친다

### Households
- 좌측: 가정 리스트
- 우측: 가정 구조와 관계
- household 안에서 같은 이름 반복 대신 역할/연결 표시

## 디자인 기준
- white / navy / soft gold 유지
- 카드 수를 줄이고 여백을 늘린다
- 같은 이름 반복보다 대표 정보 한 번 + 연결 정보 최소 노출
- 표보다 레코드 중심
- 관리툴보다 운영 OS 느낌
