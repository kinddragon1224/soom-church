# 커리어 진단 리포트 전달 플레이북

작성일: 2026-05-04

목적: `/diagnosis/report-intake`로 들어온 상세 리포트 요청을 실제 판매/전달 가능한 수동 운영 흐름으로 만든다.

핵심 원칙:

- 기본 상품은 비동기 `AI 커리어 해석 리포트`다.
- 1:1 방향 진단은 선용의 직접 시간이 필요한 경우에만 제안한다.
- 자동 리포트 생성, 결제 연동, DB 추가 없이 수동 베타 판매를 먼저 검증한다.
- 고객에게는 "AI가 대신 판단합니다"가 아니라 "김선용의 판단 기준으로 먼저 해석해드립니다"라고 말한다.

## 1. 접수 후 10분 처리 루틴

1. `ops/diagnosis-report-requests/requests.jsonl`에서 최신 요청을 확인한다.
2. `track`, `diagnosisResultType`, `currentSituation`, `wantedOutcome`을 먼저 읽는다.
3. 아래 세 가지 중 하나로 분류한다.

- `mini`: 상황이 짧고 막힌 지점이 명확하다. 미니 리포트로 충분하다.
- `detail`: 개인 조건, 경험, 제약, 선택지가 있어 상세 리포트가 적합하다.
- `premium`: 부모-자녀 의견 충돌, 퇴사/전환 리스크, 생계/건강/지역 제약처럼 직접 판단이 필요하다.

4. 고객에게 접수 확인 메시지를 보낸다.
5. 베타 가격/전달 시간을 안내한다.
6. 수동 리포트 초안을 작성한다.

## 2. 접수 확인 메시지

### 공통

```txt
안녕하세요. 숨 커리어 해석 리포트 요청 확인했습니다.

남겨주신 내용을 보니, 단순 직업 추천보다 지금 막힌 이유와 다음 행동을 먼저 정리하는 게 더 중요해 보입니다.

제가 먼저 내용을 읽고,
1) 미니 리포트로 충분한지
2) 상세 리포트가 맞는지
3) 1:1로 직접 봐야 할 정도로 복잡한지
확인해서 안내드릴게요.
```

### 학생/부모

```txt
아이 진로는 직업명을 빨리 정하는 것보다, 오래 버틸 환경과 탐색 순서를 잡는 게 먼저입니다.

남겨주신 내용을 기준으로 아이에게 맞는 환경, 부모가 피해야 할 개입, 다음 탐색 순서를 중심으로 볼게요.
```

### 20대/첫 커리어

```txt
지금은 경험이 없는 문제가 아니라, 경험이 시장 언어로 번역되지 않은 상태일 수 있습니다.

남겨주신 전공/경험/프로젝트를 기준으로 어떤 직무 언어로 바꾸면 좋을지 먼저 보겠습니다.
```

### 40-50대/전환

```txt
전환은 기존 경력을 버리는 문제가 아니라, 다시 불러낼 언어를 찾는 문제에 가깝습니다.

남겨주신 경력과 제약을 기준으로 가져갈 경험, 줄일 선택, 새로 실험할 방향을 나눠보겠습니다.
```

## 3. 베타 가격 안내 문구

### 미니 리포트 안내

```txt
현재 내용은 미니 리포트로 먼저 받아보셔도 충분해 보입니다.

미니 리포트 베타는 9,900~19,000원 가격대로 테스트 중이고,
구성은 한 줄 진단, 핵심 막힘, 다음 행동 3개입니다.

원하시면 이번 요청은 미니 리포트 기준으로 먼저 정리해드릴게요.
```

### 상세 리포트 안내

```txt
남겨주신 상황은 단순 조언보다 상세 리포트가 더 적합해 보입니다.

상세 리포트 베타는 39,000~59,000원 가격대로 테스트 중이고,
개인 조건, 피해야 할 착각, 먼저 볼 방향 2~3개, 7/14일 실행안을 포함합니다.

아직 자동 결제는 붙이지 않았고, 베타 요청 확인 후 수동으로 안내드립니다.
```

### 프리미엄 1:1 안내

```txt
이 경우는 리포트만으로도 정리할 수 있지만, 선택의 리스크가 커서 1:1 방향 진단이 더 맞을 수도 있습니다.

다만 1:1은 선용의 직접 시간이 들어가는 제한 옵션이라 기본 상품으로 열어두지는 않습니다.
먼저 상세 리포트로 정리한 뒤, 그래도 남는 선택이 있으면 1:1을 연결하는 순서를 추천드립니다.
```

## 6. 트랙별 리포트 작성 포인트

### student_parent

반드시 포함:

- 아이가 오래 버틸 가능성이 있는 환경.
- 부모가 피해야 할 개입 2개.
- 과목/활동/전공/직업을 바로 연결하지 말고 탐색 순서로 정리.
- 부모가 아이와 나눌 질문 3개.

주의:

- "이 직업이 맞다"처럼 단정하지 않는다.
- 부모 불안을 자극해 판매하지 않는다.
- 아이의 성향을 부모의 희망과 분리해 읽는다.

### early_career

반드시 포함:

- 경험을 시장 언어로 바꾼 문장.
- 아직 부족한 증거.
- 2주 안에 만들 수 있는 포트폴리오/이력서 증거.
- AI를 붙일 수 있는 반복 업무 또는 글쓰기 루틴.

주의:

- "스펙을 더 쌓으세요"로 끝내지 않는다.
- 직무명을 너무 빨리 확정하지 않는다.
- 작은 증거를 만드는 행동으로 마무리한다.

### transition

반드시 포함:

- 기존 경력의 다시 부를 이름.
- 가져갈 경험, 줄일 일, 새로 실험할 일.
- 전환 후보 2~3개.
- 피해야 할 재취업 착각 2개.

주의:

- 기존 경력을 실패처럼 말하지 않는다.
- 새 자격증/교육으로 도망가게 만들지 않는다.
- 생계, 시간, 체력, 지역 제약을 반드시 반영한다.

## 7. 1:1 방향 진단 제안 기준

1:1을 제안해도 되는 경우:

- 부모와 아이의 방향이 강하게 충돌한다.
- 퇴사, 전직, 창업처럼 비용이 큰 선택이 걸려 있다.
- 40-50대 전환에서 생계/지역/건강 제약이 크다.
- 고객이 "혼자서는 결정이 어렵다"고 직접 표현했다.
- 리포트만으로 다음 행동이 나오기 어렵다.

1:1을 제안하지 않는 경우:

- 질문이 아직 너무 넓다.
- 단순 정보 탐색 단계다.
- 미니 리포트로 다음 행동 3개가 충분히 나온다.
- 고객이 가격에 매우 민감하고 긴급도가 낮다.

## 8. 전달 메시지

```txt
리포트 초안 정리해서 보내드립니다.

이번 리포트는 정답을 확정하기보다,
지금 막힌 이유와 다음 행동을 좁히는 용도입니다.

읽어보시고 특히 1) 맞는 것 같은 부분, 2) 아닌 것 같은 부분, 3) 더 현실적으로 보고 싶은 부분을 알려주시면
다음 상품/1:1 연결 여부를 더 정확히 판단할 수 있습니다.
```

## 9. 품질 체크리스트

리포트를 보내기 전 아래를 확인한다.

- 고객이 적은 표현을 최소 2개 이상 반영했는가?
- 결과가 일반론처럼 보이지 않는가?
- 피해야 할 착각이 구체적인가?
- 다음 행동이 7일 안에 가능한가?
- 1:1 제안이 과하게 앞서 있지 않은가?
- 선용의 판단 기준이 느껴지되, 스펙 자랑처럼 보이지 않는가?

## 10. 다음 자동화 후보

아직 만들지 않는다. 단, 수동 리포트 3~5건 후 아래를 검토한다.

- 리포트 초안 생성 프롬프트.
- 리드 목록 요약 스크립트.
- 결제 링크 연결.
- 리포트 전달 이메일 템플릿.
- 관리자용 요청 목록 화면.
