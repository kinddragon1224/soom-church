# Launch Checklist

- Last updated: 2026-03-12
- Scope: MVP admin hub launch readiness

## 목적

이 문서는 "배포 가능"이 아니라 "사용자에게 런칭 가능한가"를 판단하기 위한 체크리스트다.

## 1. Product Scope Freeze

- `PRD.md`의 MVP 범위가 팀 내에서 합의되어 있다.
- 런칭 범위에서 제외할 기능이 명시되어 있다.
- 신청 도메인의 서비스 경계가 현재 상태 기준으로 합의되어 있다.

## 2. Functional QA

- 로그인/로그아웃이 정상 동작한다.
- 대시보드가 로드되고 핵심 KPI가 표시된다.
- 교인 목록/상세/등록/수정이 정상 동작한다.
- 교구/목장 배정이 정상 반영된다.
- 신청 폼 조회/생성/신청 생성/상태 변경이 동작한다.
- 공지 생성 및 조회가 동작한다.
- 활동 로그 화면이 동작한다.
- 플랫폼 관리자 화면이 최소 조회 기능을 제공한다.

## 3. Data Integrity

- 핵심 운영 데이터에 `churchId` 스코프가 적용되어 있다.
- 교인 삭제 정책이 의도한 방식으로 동작한다.
- 신청 상태 변경 후 목록과 요약 수치가 일치한다.
- seed 데이터가 데모 및 QA에 충분하다.
- 신청 도메인 중복 저장이나 불일치 현상이 없는지 확인했다.

## 4. Reliability

- `npm run build`가 통과한다.
- `npm run typecheck`가 통과한다.
- Core와 Application 서비스가 재시작 후 정상 기동한다.
- 주요 페이지가 빈 데이터 상태에서도 깨지지 않는다.
- 주요 페이지가 데이터 존재 상태에서도 정상 동작한다.

## 5. Security and Access

- 운영용 `JWT_SECRET`이 개발 기본값이 아니다.
- 배포 환경의 비밀 값이 저장소에 포함되지 않는다.
- 플랫폼 관리자 접근 대상 계정이 명확하다.
- 일반 교회 관리자와 플랫폼 관리자 접근 경계가 확인됐다.

## 6. Operations Readiness

- 환경 변수 목록이 최신 상태다.
- 배포 절차가 문서화되어 있다.
- 장애 시 담당자와 대응 순서가 정리되어 있다.
- DB 백업 또는 복구 지점이 준비되어 있다.
- 최소한의 로그 확인 방법이 정리되어 있다.

## 7. Go/No-Go Questions

- 실제 교회 운영자가 첫 주 업무를 이 제품으로 처리할 수 있는가
- 핵심 업무가 스프레드시트 없이도 가능한가
- 장애가 발생해도 복구 가능한 운영 기준이 있는가
- 아직 없는 기능보다 현재 있는 기능의 안정성이 더 높은가

## 8. Known Pre-Launch Gaps

- 권한 모델 세분화는 아직 제한적이다.
- core-service API는 외부 계약보다 내부 구현 중심이다.
- 신청 도메인 완전 분리는 끝나지 않았다.
- 향후 모듈은 런칭 범위에 넣으면 안 된다.

## Launch Rule

아래 중 하나라도 실패하면 런칭 보류로 본다.

- 로그인 또는 대시보드 진입 실패
- 교인 관리 주요 흐름 실패
- 신청 상태 변경 실패
- 멀티테넌트 데이터 경계 불일치
- 롤백 불가 상태
