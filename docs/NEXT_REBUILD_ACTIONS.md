## 현재 기준
- 실사용 앱 개발 기준 경로는 `app/[churchSlug]/*`
- 데모/마케팅 경로 `workspace/*` 와 혼동하지 않는다.
- 현재 1차 적용 완료:
  - dev 워크스페이스 구조 홈 리빌드
  - 29명 가상 시드 추가
  - React Flow 기반 노드 맵 1차 적용

## 지금까지 결정된 제품 구조
- 초기 타깃: 소형교회 / 개척교회 / 약 30~100명
- 메인 구조: `개인 - 가족(없을 수 있음) - 목장 - 교구 - 전교인`
- 메인트리와 오버레이를 분리한다.
  - 메인트리: 교구 / 목장 / 가족 / 개인
  - 오버레이: 부서 / 사역 / 상태 / 심방 / 후속 / 출석 / 헌금
- 시각화 결과는 리스트가 아니라 구조가 먼저 보여야 한다.
- 웹 우선 개발하되 모바일이 망가지지 않게 데이터/표현을 분리한다.

## 현재 코드 상태

## 다음 작업 우선순위

### 1. 용어 정리
- `목장`, `교구`, `가정`, `전교인`, `부서`, `사역`의 용어 기준 문서화
- 대흥교회식 구조와 일반화 가능한 구조를 분리해서 기록

### 2. 시각화 구조 리팩터링
- 데이터 shape와 표현 컴포넌트 분리 유지
- 웹 노드 맵과 모바일 구조 브라우저를 별도 컴포넌트로 분리
- 페이지는 두 표현을 선택해서 렌더링만 하게 유지

### 3. 구조 홈 2차
- 목장 클릭 시 우측 상세 패널 추가
- 가족 클릭 시 구성원 리스트 강화
- 개인 클릭 시 상세 페이지 이동 강화

### 4. 오버레이 추가
- 후속관리
- 심방 필요
- 출석
- 헌금
- 부서/사역 오버레이

### 5. 온보딩 연결
- Google 로그인 후 질문 흐름 정리
- 초기 입력값으로 구조 자동 생성
- 온보딩 결과가 바로 시각화 홈으로 이어지게 설계
