# MOBILE WORLD UI LOCK (2026-04-14)

이 문서는 모바일 월드 화면 개발 기준을 잠그는 고정 원칙이다.
앞으로 UI/UX 변경은 아래 원칙을 벗어나지 않는다.

## 0) 목적
- 목표: 목자가 "한 사람 선택 -> 모라에게 명령 -> 실행/기록 반영"을 가장 빠르게 수행하게 한다.
- 기준: 화려함보다 실행 속도, 설명보다 즉시 행동.

## 1) 화면 구조 고정 (필수)
- 상단 55~60%: 월드 시각 영역 (Safe Visual Zone)
  - 디오라마/상태를 보여주되 조작 요소 최소화
  - 데이터 카드는 월드를 가리지 않게 작고 반투명
- 하단 40~45%: 조작 영역 (Interactive Zone)
  - 모라 명령창, 최근 출력, 입력창/전송, 운영 요약
- 하단 탭바는 고정, safe-area 기준으로 겹침 금지

## 2) 언어/의미 체계 고정
- 금지: 전투력, 랭킹, 레어도, 경쟁 유도 카피
- 사용: 목양, 돌봄, 후속, 심방, 기도, 운영 지수, 오늘 실행
- 개인 비교 대신 공동체 운영 피드백을 우선

## 3) 상호작용 고정
- 채팅은 대화용이 아니라 명령 인터페이스
- 프리셋 칩은 입력 보조 (짧고 빠르게)
- 출력창은 최근 실행 결과 확인용 (짧은 로그 중심)
- 입력창은 과도한 높이 금지 (월드 영역 침범 금지)

## 4) 시각 설계 고정
- 월드가 메인, 패널은 보조
- 과한 오버레이/임시 마커/의미 없는 장식 금지
- 에셋은 레이어 분리 품질 우선 (배경/건물/오브젝트/캐릭터)

## 5) 게이미피케이션 적용 원칙
- 허용: 진행 가시화, 자동화 보상, 상태 피드백
- 금지: 사람 간 경쟁/서열화
- 보상은 "목장 전체의 개선"으로 표현

## 6) AFK 운영 원칙
- 앱 재진입 시 모라가 먼저 핵심 변화 요약
- 사용자가 데이터 탐색하지 않아도 다음 행동이 바로 보이게 구성

## 7) 실행 규칙 (개발 협업)
- 변경 전에 "이 변경이 상단55~60/하단40~45 구조를 깨는가" 먼저 확인
- 변경 후 모바일 실기기 기준 스크린샷으로 즉시 검증
- 전달 명령은 한 줄 고정:
  - `cd /home/kinddragon/.openclaw/workspace/soom-church && npm run mobile:view`
