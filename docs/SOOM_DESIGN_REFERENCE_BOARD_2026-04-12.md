# SOOM Design Reference Board — 2026-04-12

## 1. 한 줄 기준
숨은 `따뜻한 목장 운영 OS`다.

- 위에는 장면이 보이고
- 아래에는 입력이 있고
- 사람은 카드보다 관계와 공간으로 먼저 느껴져야 한다.

---

## 2. 월드 레퍼런스 조합

### 기본 조합
- Cozy Grove 60%
- Pokémon village structure 25%
- Spiritfarer emotional tone 15%

### 월드가 가져야 할 느낌
- 따뜻한 손그림 마을
- 저녁빛, 크림, 초록, 나무색 중심
- 경쟁형 게임이 아니라 돌봄형 공간
- 관리 툴보다 머물고 싶은 운영 공간

### 월드 금지 요소
- 관리자 대시보드처럼 보이는 카드 나열
- 텍스트가 화면을 지배하는 구조
- 스티커처럼 뜨는 NPC 합성
- 과한 HUD, 과한 버튼, 과한 배지

---

## 3. NPC 기준

### 역할
- NPC는 기능 버튼이 아니라 공간 주민이자 안내자다.
- 요한, 마리아는 `목양 공간으로 안내하는 따뜻한 존재`여야 한다.

### 시각 기준
- 집보다 작아야 한다.
- 교회 앞 길 위에 발이 붙어 있어야 한다.
- 바닥 그림자와 톤 보정이 필수다.
- 배경에 박아 넣는 것이 아니라 월드 위에 자연스럽게 서 있어야 한다.

### 상호작용 기준
- hover는 미세하게만
- 클릭 가능성은 느껴지되 UI 버튼처럼 보여서는 안 된다.
- 이름표는 최소화하고, 필요할 때만 드러난다.

---

## 4. 채팅 UI 기준

### 레퍼런스 방향
- shadcn minimal AI chat 계열
- 가운데 집중형 입력 구조
- 설명보다 바로 입력하게 만드는 구조

### 숨에 맞는 형태
- 큰 입력창 하나
- 짧은 placeholder 하나
- 하단 태그 몇 개만 유지
- 보조 패널, 설명 카드, 우측 카드 최소화

### 채팅 금지 요소
- 기능 설명이 많은 화면
- 입력보다 카드가 더 눈에 띄는 구조
- 월드를 덮는 오버레이 입력 UI

---

## 5. 레이아웃 기준

### 홈
- 오늘 어디서 시작할지 바로 보여주는 허브
- Chat / World / Records가 연결되지만 복잡하지 않아야 함

### 월드
- 장면이 먼저 보여야 함
- 입력창은 이미지 아래에 위치
- NPC 클릭으로 공간 진입

### 목양 공간
- 사람 중심 운영실
- 상태, 기록, 다음 행동이 이어지는 구조
- 배경 퀄리티에 맞는 UI 톤 필요

---

## 6. 색감 기준
- Cream
- Warm beige
- Muted green
- Evening gold
- Soft brown
- Dusty rose / muted blue (NPC 포인트용)

### 피해야 할 색감
- 차가운 SaaS 블루 중심
- 형광톤 강조색
- 검정/회색 위주 대시보드 톤

---

## 7. UI 톤 기준
- 둥근 모서리
- 얇은 경계선
- 유리처럼 과한 블러보다 종이/나무/따뜻한 패널 감각
- 말 수 적은 인터페이스
- 자연어 처리 느낌보다 운영 행동 중심 문장

---

## 8. 벤치마킹 -> 우리화 대응표

| 참고 UX 요소 | 숨에서 쓰는 운영 용어 | 적용 위치 |
| --- | --- | --- |
| HUD / status bar | 오늘 목양 메인 / 오늘 운영 지표 | 월드 상단 |
| quest / mission | 오늘 실행안 / 실행 항목 | 기록 탭, 월드 버튼 |
| roster | 목원 관리 목록 | 목원 탭 |
| unit stats | 돌봄 / 기도 / 후속 상태 | 목원 상세 |
| command input | 모라 명령창 | 월드 중단 |
| activity log | 목양 기록 / 자동 반영 로그 | 기록 탭 |

## 9. 구현 우선순위
1. 월드/NPC 자연스러움
2. 채팅 입력 미니멀 정리
3. 목양 공간 UI 고급화
4. 홈 구조와 월드 연결 강화
5. 이후 실제 데이터 연결
